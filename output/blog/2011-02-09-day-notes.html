Day notes - Wed 9th Feb 2011

Mongoid doesn’t support migrations out of the box. Although it has a rake task to create indexes on collections, there is no nice way to remove them. There is mongoid_rails_migrations but that doesn’t work with mongoid releases after 2.0.0.rc.1 (see issue 7). One solution is to create a standard active record migration and do something like the example in this gist.

I started working on GFRs new project this afternoon. I got set-up and updated the developer set-up notes in the process. Wrote an email about the merits of either migrations and/or having an up-to-date schema.

Watched Mr Nice - it was good.
